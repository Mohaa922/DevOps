Introduction à DevOps

DevOps est une approche qui combine le développement logiciel (Dev) et les opérations informatiques (Ops) pour améliorer la collaboration, automatiser les processus et accélérer la livraison logicielle.

« DevOps est une culture, un mouvement, ou une pratique qui vise à unir le développement logiciel et les opérations. »

1. Git - Contrôle de version

Git est un système de contrôle de version distribué qui permet de suivre les modifications du code et collaborer efficacement.

Commandes de base Git :

git init
git clone <url>
git status
git add <fichier>
git commit -m "message"
git push
git pull

Explications :

  • git init : Initialise un nouveau dépôt Git.
  • git clone <url> : Copie un dépôt distant localement.
  • git status : Montre l'état des fichiers dans votre dépôt local.
  • git add <fichier> : Prépare des fichiers pour le commit.
  • git commit -m "message" : Enregistre les changements avec un message.
  • git push : Envoie vos commits vers un dépôt distant.
  • git pull : Récupère les changements d'un dépôt distant.

2. Docker - Conteneurisation

Docker permet de créer, déployer et exécuter des applications dans des conteneurs légers et portables.

Commandes de base Docker :

docker build -t nom_image .
docker images
docker run -it nom_image
docker ps
docker stop <container_id>
docker rm <container_id>
docker rmi <image_id>

Explications :

  • docker build : Crée une image Docker à partir du Dockerfile.
  • docker images : Liste les images locales.
  • docker run : Exécute un conteneur basé sur une image.
  • docker ps : Affiche les conteneurs en cours d'exécution.
  • docker stop : Arrête un conteneur en cours.
  • docker rm : Supprime un conteneur arrêté.
  • docker rmi : Supprime une image Docker locale.

3. Jenkins - Intégration Continue

Jenkins est un serveur d'automatisation open-source pour réaliser l'intégration continue et la livraison continue (CI/CD).

Commandes utiles Jenkins (via CLI ou pipeline) :

java -jar jenkins-cli.jar -s http://localhost:8080/ build nom_job
java -jar jenkins-cli.jar -s http://localhost:8080/ list-jobs
java -jar jenkins-cli.jar -s http://localhost:8080/ get-job nom_job

Dans les pipelines Jenkins, on utilise souvent la syntaxe Groovy pour définir stage, steps et automatiser les builds.

4. Kubernetes - Orchestration de conteneurs

Kubernetes automatise le déploiement, la gestion et la mise à l'échelle des applications conteneurisées.

Commandes de base kubectl :

kubectl cluster-info
kubectl get nodes
kubectl get pods
kubectl describe pod <nom_pod>
kubectl logs <nom_pod>
kubectl apply -f fichier.yaml
kubectl delete pod <nom_pod>

Ces commandes permettent de consulter l'état du cluster, des pods, appliquer des configurations et consulter les logs.

5. Ansible - Automatisation de la configuration

Ansible simplifie la gestion et l'automatisation des configurations sur plusieurs serveurs.

Commandes de base Ansible :

ansible all -m ping
ansible-playbook playbook.yml
ansible -m shell -a "uptime" nom_serveur
ansible -i hosts all -m yum -a "name=httpd state=present"

Explication : ansible-playbook exécute des scripts de provisioning nommés playbooks.

6. Terraform - Infrastructure as Code

Terraform permet de définir et gérer votre infrastructure via des fichiers de configuration déclaratifs.

Commandes de base Terraform :

terraform init
terraform plan
terraform apply
terraform destroy
terraform fmt
terraform validate

Ces commandes initialisent un projet, prévoient les changements, appliquent, détruisent et valident la configuration.
